# AGENTS.md — правила для AI-агентов в этом репозитории

Работает для: Codex CLI, Claude Code, Cursor, Copilot и любых других агентов.
Нормативный документ проекта — **PLAN.md** (архитектура, фазы, схемы, промпты). Этот файл — правила поведения.

---

## О проекте

**«Фразодром»** — открытый некоммерческий тренажёр английского для русскоязычных: перевод предложений RU→EN по грамматическим паттернам (A1–C1), интервальные повторения (FSRS), listening/speaking, экзамены, ИИ-репетитор по собственному API-ключу пользователя (BYOK). Веб-PWA без бэкенда: весь прогресс в браузере, хостинг — GitHub Pages, данные — открытый корпус Tatoeba (CC-BY).

Стек: `app/` — Vite + React + TypeScript + Tailwind + Zustand + Dexie + ts-fsrs; `pipeline/` — Python 3.12 + uv + polars + spaCy (CLI `etr`); `packs/` — сгенерированные JSON-паки курса (коммитятся).

---

## Поведение

1. **Думай перед кодом.** Неоднозначность → назови варианты и спроси, не выбирай молча. Есть более простой путь — скажи об этом.
2. **Простота прежде всего.** Минимум кода, решающего задачу. Ничего «на вырост»: ни абстракций для одноразового кода, ни фич, которых не просили. Тест: «сеньор скажет, что это переусложнено?» — упрощай.
3. **Хирургические изменения.** Трогай только то, что нужно для задачи. Не рефактори соседний код, не «улучшай» чужие комментарии и форматирование. Заметил мёртвый код — упомяни, не удаляй.
4. **Проверяемые критерии.** Любая задача превращается в проверку: «добавь валидацию» → «напиши тест на невалидный ввод и сделай его зелёным». У каждой фазы PLAN.md §16 есть acceptance-критерии — прогоняй их.
5. **Не перечитывай весь репозиторий** «для понимания» — точечные чтения. Решения храни в файлах (PLAN.md, MEMORY.md), а не в чате.

## Рабочий процесс

- **Одна фаза = одна сессия.** Фазы и их критерии — PLAN.md §16. Правила работы с планом — PLAN.md §0. Будущие фазы не реализуем.
- В начале сессии прочитай **MEMORY.md** (рабочий контекст), затем PLAN.md §0 и §16.
- После фазы: прогнать acceptance-критерии, отметить чекбокс в PLAN.md §16, дописать 3–5 строк в журнал в конце PLAN.md, обновить MEMORY.md (только подтверждённые факты и решения, кратко).
- Схемы JSON в PLAN.md нормативные: меняешь схему — фиксируй изменение в PLAN.md и журнале.
- Устаревшие внешние факты (лимиты API, версии) — смотри PLAN.md §17.2; если данных нет, попроси пользователя прогнать готовый запрос из §17.3. Сам в интернете не ищи без запроса.

## Конвенции

- **Языки:** код, комментарии, коммиты, docstrings — английский. Строки UI — только через i18n-словарь, полное дублирование ru+en, хардкод строк запрещён. Общение с пользователем — русский.
- **Коммиты:** Conventional Commits, только заголовок без body (`feat(drill): add hint ladder`). Коммитить часто, мелкими логическими шагами. Не упоминать AI-инструменты в коммитах.
- **Перед коммитом:** `app/` — eslint + tsc + vitest; `pipeline/` — ruff + mypy + pytest; трогал паки — `uv run etr validate`.
- **Тесты:** любой баг чекера сначала фиксируется кейсом в табличных тестах, потом чинится. Новая логика — сначала тест.
- **Money-эквивалент проекта:** нормализация ответов (`normalize()`) — единственный источник правды для сравнения строк; никаких ad-hoc `.toLowerCase()` сравнений в обход неё.

## Запрещено (NEVER)

- Коммитить секреты, API-ключи, `.env` — **репозиторий публичный**.
- Добавлять платные API, бэкенд, аккаунты, телеметрию/аналитику — без явного «да» пользователя.
- Генерировать основной контент курса LLM'ом в рантайме — база курса собирается пайплайном из корпуса (PLAN.md §1.4).
- Хардкодить лимиты провайдеров в код — только конфиг (PLAN.md §8).
- Дёргать публичный API languagetool.org автоматическими запросами (только self-hosted, PLAN.md §13.3).
- Создавать новые .md-файлы без запроса пользователя.
- Ломать стабильные id навыков и схему паков без фиксации в PLAN.md.
- Убирать атрибуцию Tatoeba (CC-BY) — страница «Данные и лицензии» обязательна.
